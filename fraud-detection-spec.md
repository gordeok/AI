# GO르덕 사기 방지 AI 스펙

## 개요

단톡방 채팅 대화 흐름을 실시간으로 분석해 사기 패턴을 감지하는 시스템.
키워드 단순 매칭이 아닌 **대화 맥락 전체**를 보고 판단하며,
위험 감지 시 채팅방 전체에 사기 주의 배너를 즉시 표시한다.

---

## 탐지 구조

```
단톡방 메시지 전송
        ↓
Spring Boot WebSocket 수신
        ↓ 비동기로 FastAPI 호출
1단계: 규칙 기반 필터 (즉시, ~1ms)
        ↓ 미탐지 시
2단계: Ollama EXAONE3.5:2.4b (대화 맥락 분석, ~1~2초)
        ↓
SAFE     → 아무것도 안 함
WARNING  → 채팅방 상단 주의 배너
DANGER   → 강한 경고 배너 + 신고 버튼 강조
```

### 분석 대상

- **전체 대화 맥락 분석** (발신자 무관)
- 판매자·구매자 모두 감지 대상 — 어느 쪽이든 사기 패턴 감지 시 경고
- 최근 N개 메시지를 슬라이딩 윈도우로 묶어서 Ollama에 통째로 전달

### 경고 표시 대상

- **채팅방 전체 공개** — 단톡방 참여자 모두에게 배너 표시
- 발신자에게도 표시 ("이 문구는 사기로 오해받을 수 있어요")

---

## 1단계 — 규칙 기반 필터 (Rule-based Filter)

맥락 분석 전에 명확한 위험 패턴을 즉시 탐지한다.

**탐지 대상 패턴 예시**
- 선입금 유도 (예: "먼저 입금해주시면", "계좌로 먼저 보내주세요")
- 외부 결제 수단 유도 (예: "카카오페이로 직접", "문화상품권")
- 외부 채널 이동 유도 (예: "오픈채팅으로 와주세요", "카톡 ID 알려드릴게요")
- 안전결제 사칭 (예: "○○안전결제 링크로 결제해주세요")
- 외부 URL 포함 (예: http:// 또는 bit.ly 등 단축 URL)
- 개인정보 직접 요청 (예: "계좌번호랑 이름 알려주세요")

**동작**
- 패턴 매칭 즉시 → DANGER로 처리, 채팅방 전체에 강한 경고 배너 표시
- 백엔드에 탐지 이벤트 로깅

## 2단계 — Ollama EXAONE3.5:2.4b

### 모델 선택 근거

| 항목 | 내용 |
|------|------|
| 모델 | EXAONE3.5:2.4b (LG AI Research) |
| 비용 | 완전 무료 — 로컬 실행, API 비용 없음 |
| 한국어 성능 | 한국어 특화 모델, 구어체·분철 맥락 이해 우수 |
| 리소스 | 2.4B 파라미터, VRAM 4GB |
| 개인정보 | 로컬 추론 — 채팅 데이터 외부 전송 없음 |

> Claude API / GPT API는 비용 발생 + 채팅 데이터 외부 전송 문제로 제외

### 입력 구성

```
[시스템 프롬프트]
너는 분철 거래 사기 탐지 AI야.
아래 대화에서 사기 징후를 분석하고 JSON만 반환해.
분류 기준:
- DANGER: 선입금 유도, 외부결제 유도, 안전결제 사칭, 외부채널 이동 유도
- WARNING: 모호한 거래 조건, 비정상 가격, 불분명한 구성품 안내
- SAFE: 정상적인 분철 거래 대화

[대화]
[판매자] 포카 상태 S급이에요
[구매자] 분철비 얼마예요?
[판매자] 입금 먼저 해주시면 바로 보내드릴게요
```

### 출력 형식

```json
{
  "risk_level": "DANGER",
  "reason": "선입금 유도 문구 감지",
  "triggered_message": "입금 먼저 해주시면 바로 보내드릴게요"
}
```

### 파서 안정성 — 3단계 폴백

| 단계 | 처리 방식 |
|------|-----------|
| 1 | `json.loads`로 정상 파싱 |
| 2 | 파싱 실패 시 첫 번째 `{...}` 블록 추출 재시도 |
| 3 | 블록 추출도 실패 시 `SAFE / WARNING / DANGER` 키워드 폴백으로 결과 보장 |

## API 인터페이스 (FastAPI)

```
POST /api/fraud/analyze
```

**Request**
```json
{
  "room_id": "string",
  "messages": [
    { "sender_id": "string", "role": "판매자|구매자", "content": "string", "timestamp": "ISO8601" }
  ]
}
```

**Response**
```json
{
  "rule_triggered": true,
  "matched_patterns": ["선입금 유도"],
  "llm_risk_level": "DANGER",
  "llm_reason": "선입금 유도 문구 감지",
  "action": "show_danger_banner"
}
```

**action 값**
| 값 | 처리 |
|----|------|
| `none` | 아무것도 안 함 |
| `show_warning_banner` | 채팅방 상단 주의 배너 |
| `show_danger_banner` | 강한 경고 배너 + 신고 버튼 강조 |

## 테스트 시나리오

> 모델 배포 후 회귀 테스트 기준으로 사용.

| 구분 | 예시 문구 | 기대 결과 |
|------|-----------|-----------|
| 정상 거래 | "포카 상태 괜찮나요?", "직거래 가능한가요?" | `SAFE` |
| 선입금 유도 | "입금 먼저 해주시면 바로 보내드릴게요" | `DANGER` |
| 외부 채널 이동 | "카카오톡 오픈채팅으로 와주세요" | `DANGER` |
| 안전결제 사칭 | "○○안전결제 링크로 결제해주세요" | `DANGER` |
| 외부 결제 링크 | "이 링크 들어가서 결제해주세요 (http://...)" | `DANGER` |
| 시세 대비 급매 | "원가 10만원인데 2만원에 급처합니다" | `WARNING` |
| 개인정보 요청 | "계좌번호랑 이름 알려주세요" | `DANGER` |
| 모호한 거래 조건 | "조건은 나중에 얘기해요" | `WARNING` |
| 문화상품권 유도 | "문화상품권으로 결제해주시면 돼요" | `DANGER` |

## 민감 정보 자동 처리

분철 단톡방(1:N 그룹 채팅)에서 민감 정보 노출을 방지한다.

- **주소·운송장 번호 자동 마스킹**: 단톡방 내 감지 시 `서울시 ○구 *로` 형태로 마스킹 후 경고 안내
- **1:1 채널 강제 분리**: 운송장 번호·배송 주소는 개설자↔참여자 간 1:1 채널로만 전송, 단톡방 입력 차단

## 비 AI 사기 방지 시스템 연동

| 기능 | 설명 |
|------|------|
| 즉각 신고 | 증거 이미지 첨부 필수, 채팅 로그 자동 첨부 |
| 신고 즉시 거래 정지 | 신고 접수 시 해당 사용자 거래 자동 정지 |
| 사기 의심 계정 공개 | 커뮤니티 사기 게시판에 공시 |
| 안전거래 유도 | 외부 링크 패턴 감지 시 자동 경고 |

## 기술 스택

| 항목 | 내용 |
|------|------|
| LLM | EXAONE3.5:2.4b — Ollama 로컬 서빙, 무료 |
| 규칙 필터 | Python 정규식 기반 하드코딩 키워드 |
| 서빙 | Python FastAPI (마이크로서비스) |
| 연동 | Spring Boot 백엔드 → FastAPI 비동기 호출 |
| 실시간 | WebSocket STOMP 메시지 수신 후 분석 트리거 |
